While implementation remained the main focus, this week also introduced a new aspect of the project.

We had a dedicated meeting about the digitalization of the membership benefits PDF. During this session, the concept was revisited and explained in more detail, including how the digital version is imagined and how it should function within the website.

The goal of this transformation is to move away from a static PDF and create a more interactive, structured, and user-friendly experience.

During the meeting, I focused on understanding:

- How the digital version should work  
- How the content could be reorganized  
- What type of user experience is expected  
- How this feature fits into the overall membership journey  

At this stage, my role was mainly to listen, understand the concept, and begin thinking about how UX principles could be applied to this transformation.

Overall, this week required balancing continued implementation with understanding a new project direction.

This week required:

- Practical WordPress implementation skills  
- Attention to detail and consistency  
- Problem-solving and adaptability  
- Basic HTML and CSS understanding  
- Active listening and concept understanding

## 3) What did I learn this week? In what way?

This week reinforced how implementation becomes smoother with experience, while also introducing the challenge of understanding new UX concepts.

I learned that:

- Implementation becomes more efficient as familiarity with tools increases  
- Less trial and error is needed when the underlying logic is understood  
- UX projects often involve working on multiple phases at the same time  
- Understanding a concept deeply is necessary before proposing solutions  
- Transforming static content into digital experiences requires rethinking structure and interaction
