## 1) What have I been doing this week? What skills did I need?

In Week 9, I focused on refining the sections I had already implemented in WordPress, improving visual consistency, and adding more dynamic elements to the page.

A key part of this week was enhancing the interface by introducing motion. I worked on adding subtle animations to make the page feel more interactive and engaging, while still respecting the overall design and user experience.

One of the main technical challenges this week was implementing a moving logo strip. Since there was no pre-built widget available in Elementor for this functionality, I initially attempted to create it without code, relying only on built-in tools.

Although I partially succeeded, I encountered limitations—especially with creating a seamless looping animation. The strip would not loop correctly, which affected the overall experience.

After several attempts and increasing frustration with the limitations of the no-code approach, I decided to implement the component manually using HTML and CSS. This approach proved to be significantly more flexible and efficient. By writing the code from scratch, I was able to fully control the animation behavior and achieve the desired looping effect without constraints.

This experience showed me that while no-code tools are useful, they can become limiting for more advanced interactions, and knowing when to switch to code is essential.
